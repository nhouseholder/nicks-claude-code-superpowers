# Never Give Up — But Be Smart About It

## The Evidence Gate

Before deciding to persist or move on:

> **Is there independent evidence this idea has value?**

| Evidence | Action |
|----------|--------|
| Proven profitable in isolation, backed by data | Fight for it — integration problem is solvable |
| Untested theory, no data | Test in isolation first |
| 3+ well-designed attempts failed | Shelve with detailed notes |

## Token Discipline

**3 same-approach attempts, then escalate or change strategy:**
1. Full implementation + backtest. Diagnose on failure.
2. Different approach informed by attempt 1.
3. Still failing? STOP. Present findings to user, let them decide.

Genuinely different strategies reset the cap. Same approach + different numbers = not learning.

**Signs to STOP:** Same approach with tweaked numbers. Each attempt takes longer. Can't articulate why the next try should work differently.

**Signs to KEEP GOING:** Each failure teaches something specific. Concrete hypothesis for next approach. User explicitly asked to continue.

## What "Trying Harder" Means

NOT: same code with different magic numbers.
YES: fundamentally different approach (additive→conditional, linear→sigmoid, individual→joint optimization). Each retry must diagnose the prior failure first.

## Rules

1. Evidence gates persistence
2. 3 same-approach attempts then escalate
3. Each retry must be fundamentally different
4. Log lessons, not labels — never "X is a failed feature"
5. Preserve baseline always — every attempt safely revertible
6. Diagnose before retrying
