## 5. Warnings
- **PLAN_AUTO_SWITCH_IMPOSSIBLE anti-pattern is load-bearing** — `~/.claude/anti-patterns.md` now documents the 10-commit history of failed attempts to auto-switch the running session's model from a hook. Claude Code hooks architecturally cannot do this. Do NOT re-attempt: `get_current_model()` reads, settings.json writes to the "model" key, or any "claim to know which model is running" message. The running session model is locked at startup; settings.json writes only affect the NEXT session.
- **Cross-project guard bleed was the specific root cause of the false-fire** — `~/.claude/.plan-guard-active` is a global file but guards should be project-scoped. The fix uses cwd comparison against the guard's stored content. If you ever add a new guard type, remember to project-scope it from day one.
- **Plan-intent detection is too loose to lock tools** — conversational phrases ("make a plan", "write me a plan", "plan mode") are fine for injecting the plan format instructions, but they are NOT strong enough signals to create a guard file that blocks Edit/Write. Only the real `ExitPlanMode` tool call is. This was the second-order root cause.
- **Test harness caveat** — earlier verification run destroyed plan files mid-test because the plan-intent branch cleans `PLAN_DIR`. Fixed by using a dummy plan file that the test re-creates per scenario. If you write more hook tests, beware of side effects that mutate the state being tested.
